Expand description

Enhanced execution context with hierarchical support

Fields§

§id: String

Unique context ID

§parent_id: Option<String>

Parent context ID for hierarchical reference (avoids circular references)

§scope: ContextScope

Context scope

§inheritance: InheritancePolicy

Inheritance policy

§conversation_id: Option<String>

Current conversation ID

§user_id: Option<String>

User ID if applicable

§session_id: Option<String>

Session ID for tracking

§data: HashMap<String, Value>

Local context data

§shared_memory: SharedMemory

Shared memory for inter-agent communication (transient data)

§state: Option<Arc<dyn StateAccess>>

State access for persistent data (first-class citizen for component communication) Uses StateAccess trait to avoid direct dependency on llmspell-state-persistence

§events: Option<Arc<dyn EventEmitter>>

Event emitter for component lifecycle events (observability and coordination) Uses EventEmitter trait to avoid direct dependency on llmspell-events

§metadata: EventMetadata

Event metadata for correlation

§security_context: Option<SecurityContext>

Security context (placeholder for Phase 3.2 integration)
